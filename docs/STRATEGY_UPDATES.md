# 策略更新记录

本文是策略完整更新的追加式审计日志。条目按时间从旧到新排列；每次完成“策略或执行逻辑修改、基础与压力回测、全量测试、文档同步”闭环后，只在文件末尾追加新版本，不覆盖或重写既有版本结论。

## 版本规则

- 版本格式为 `MAJOR.MINOR.PATCH`。
- `MAJOR`：账本、执行接口或策略含义发生不向后兼容的变化。
- `MINOR`：完成一轮影响信号、仓位、风控或回测结果的完整策略更新。
- `PATCH`：不改变交易决策的工程修复、测试或文档更正。
- 权威版本位于 `src/etf_rotation/version.py`；`pyproject.toml` 与 `configs/strategy.yaml` 必须保持一致，自动测试负责校验。
- `execution.strategy_tag` 用于 QMT 委托和本地账本归属，不是策略版本号；常规升级不得随版本号自动改动它。
- 每个版本条目必须记录目标、代码与配置变化、回测对比、被拒候选、验证结果和已知限制。
- 可运行 `etf-rr --version` 查询当前权威版本；回测摘要和信号 JSON 也会携带该版本。

## v0.3.0 — 2026-07-13

### 更新目标

- 在不提高最大风险上限的前提下，提高可复现收益和风险调整后收益。
- 先修复回测与实盘一致性缺陷，再比较少量具有经济含义的候选。

### 数据与回测修复

- 校正 QMT 前复权序列遗漏的明确 ETF 份额拆分：513100.SH 约 5:1、513500.SH 约 2:1。
- 开盘跳空止损改为先于例行调仓执行，并禁止同一标的在同一开盘立即买回。
- 完整退出和新建仓不再被 1% 免交易带拦截。
- `live-once` 在周度计划前先评估并锁存实时风险退出，避免实盘重新打开已触发止损的标的。

### 策略改进

- 风险调整排名从 $M/\sigma$ 改为 $M/\sigma^{0.75}$，减轻排名与逆波动率仓位的重复风险惩罚。
- ATR 初始止损和跟踪止损增加 1.5% 最小距离，减少低波动资产的噪声止损。
- 闲置风险预算可配置最多 30% 的国债 ETF，但仅在长期趋势和绝对动量均为正时启用。

### 被拒候选

- 提高单资产上限或目标波动率。
- 权重截断后强制重新分配至满仓。
- 放大 `risk_on` 仓位、取消止损或设置统一禁买冷却。

这些方案在开发、验证或近期区间中出现明显回归，未进入生产配置。

### 验证与限制

- 全量单元测试、Python 编译、安全扫描和 `git diff --check` 通过。
- 自动稳健性验证与标准回测口径一致：三倍成本 CAGR 3.58%，10 个滚动三年窗口全部为正，最差滚动 CAGR 1.73%，前缀不变性通过。
- 验证报告记录策略版本及代码、配置、行情 SHA-256 指纹，便于后续版本做可复现对比。
- 相邻参数方向总体稳定，但所有历史区间已经参与研究，不是未触碰样本。
- 仍存在现存 ETF 池幸存者偏差、跨境折溢价缺失、日线成交与流动性模型简化等限制。
- 版本应冻结后进行至少 20 个交易日向前模拟；历史回测不构成收益保证。
