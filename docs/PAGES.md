# GitHub Pages 建设评估

## 结论

本项目适合建设中文 GitHub Pages。它已经具备完整策略说明、验证数据、SVG 图示和稳定的版本记录，静态站可以明显改善移动端阅读、文档导航和图表展示。

推荐地址：

```text
https://www.guoyaohua.com/etf-adaptive-rotation-qmt/
```

账号已有个人 Pages 与 `www.guoyaohua.com` 自定义域，本项目作为子路径发布，不应在仓库中新增 `CNAME`。

## 推荐方案

采用 `site/` 原生静态页面，并通过 GitHub Actions 发布：

- 无需 Node、Ruby 或额外主题依赖；
- 能精确控制中文排版、响应式图表和项目子路径；
- 可复用 `docs/assets/` 的 SVG；
- CI 只上传静态文件，不接触 QMT、账户或本地报告。

推荐栏目：

1. 首页：项目定位、核心指标、策略流程与风险提示；
2. 策略：趋势、动量、仓位与四周错峰；
3. 验证：1×/2×/3× 成本、滚动窗口、已知偏差；
4. 上手：安装、dry-run 和模拟盘；
5. 安全：账本、幂等计划与敏感信息规范；
6. 版本：链接至追加式更新日志。

## 不能做什么

GitHub Pages 是静态托管，不能：

- 连接本机 QMT 或运行 `xtquant`；
- 在线执行真实回测；
- 保存资金账号、Token、账本或交易数据；
- 替代实盘监控程序。

若未来需要在线交互，只应使用不含个人行情和账户数据的前端参数演示，并明确标注为教学模拟。

## 启用步骤

1. 新增 `site/` 和 Pages 部署工作流；
2. 确保所有资源使用 `/etf-adaptive-rotation-qmt/` 子路径或相对路径；
3. 在仓库 `Settings → Pages` 将 Source 设为 `GitHub Actions`；
4. 首次部署后把仓库 Homepage 更新为站点地址；
5. 检查手机、深色模式、404、SEO 描述和图表口径；
6. 版本发布时只更新冻结指标，不从被忽略的本地报告自动上传账户相关内容。

## 工作量评估

- 最小站点：约半天，可完成首页、策略摘要、验证快照和安装入口；
- 完整文档站：约 1–2 天，增加多页导航、响应式图表、SEO、无障碍和自动链接检查；
- 后续维护：每个完整策略版本约 15–30 分钟同步冻结指标和版本说明。

建议先发布最小站点，再根据访问和反馈扩展；不建议一开始引入复杂前端框架。
