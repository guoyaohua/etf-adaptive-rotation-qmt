# 参考策略审查结论

审查对象：

- `ETF均线策略v3.0_实盘.py`
- `ETF趋势策略v2.0_模拟盘.py`

## 可复用部分

- QMT `xtdata` 行情入口、全推订阅与 `XtQuantTrader` 下单接口。
- 卖出优先、整手数量、延迟行情拒绝、交易标签和尾盘安全处理的工程意识。
- 使用买卖盘口而非最新价作为执行参考。

## 不直接复用的部分

1. 均线版默认实际使用开盘累计 VWAP，并非配置名所表达的固定周期均线；买入趋势判断方向与日志含义相反，初始化还可能制造虚假上穿。
2. 两套策略的“止盈价”由峰值向下计算，本质是跟踪退出；没有独立的初始硬止损、组合风险预算和净值熔断。
3. 以历史高振幅作为候选条件会系统性偏向高噪声品种，却没有成交额、折溢价、相关性或横截面强弱约束。
4. 异步下单缺少完整的 pending/filled/cancelled 状态机，多个信号可能重复下单或突破持仓上限。
5. 趋势版用不等时间间隔的 tick 数近似“一分钟”，并把累计成交量当作单 tick 成交量使用。
6. 趋势版参数搜索在同一单日样本上选择和评分，而且同一个策略状态跨 ETF 复用，存在状态污染；未计成本、滑点和未成交，结果不能作为收益证据。
7. 原模拟只覆盖固定日期或很少样本，无法回答跨年份、不同市场状态下的稳定性问题。

## 本项目的对应改造

- 信号、回测、风控、成交、QMT 适配器完全分层。
- 每个品种独立状态；所有日线信号严格延迟到下一交易日开盘执行。
- 选池先检查历史长度、价格和成交额，再做趋势、强弱、分组、相关性约束。
- 风险按波动率预算，组合回撤优先于单个信号；真实下单默认关闭。
- 验证报告同时展示基础成本、双倍成本、年度分段和明确的失败门槛。
